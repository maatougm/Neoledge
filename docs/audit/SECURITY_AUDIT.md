# NeoLeadge Security Audit

**Date:** 2026-05-19
**Auditor:** Security Reviewer Agent
**Scope:** `web/back-nest/` (NestJS 11), `web/Front/customapp/` (Vue 3), `web/Transcription/` (FastAPI), `deploy/neoleadge/` (Docker Compose prod)
**Branch audited:** `nest-back`

---

## Executive Summary

The NeoLeadge codebase demonstrates above-average security hygiene for a project of its size and complexity. Authentication (JWT algorithm pinning, token versioning, account lockout, timing-safe login, TOTP audience segregation), input validation (global `ValidationPipe` with whitelist), and access control (ProjectAccessGuard, per-route RolesGuard) are all correctly implemented. However, the audit identified three items requiring prompt action: a `.env` file containing a real JWT secret and a weak `TRANSCRIPTION_SECRET` that are tracked locally on the developer machine (not in git, but the file exists on disk and could be leaked via IDE sync or CI); two vulnerable transitive dependencies (`lodash` — RCE via template injection, `path-to-regexp` — ReDoS, `axios` prototype-pollution); and a SQL injection risk in the `backfill-embeddings` CLI script where `--project-id` is interpolated directly into a raw SQL `LIMIT` clause (the value escaping is manual quote-doubling, not parameterised). Additionally, the NestJS server container runs as root (no `USER` directive in its Dockerfile), which elevates blast radius for any container escape. Twelve more medium/low findings follow.

## Not an Issue

The following items were checked and found to be correctly implemented:

- **JWT algorithm pinning** — `algorithms: ['HS256']` is enforced in both `JwtStrategy` (verification) and `JwtModule` (signing). The `aud` claim is validated (`'access'` vs `'totp'`). TOTP temp tokens are segregated by audience and rejected by the standard auth guard. No `none`-algorithm bypass path exists.
- **Token versioning / session invalidation** — `JwtStrategy.validate()` fetches `tokenVersion` from the DB and rejects tokens whose version is behind (covers password change, account deactivation). Incremented in `changePassword()`, `resetPasswordByToken()`, and `resetPassword()` (admin).
- **Timing-safe login** — Unknown email addresses trigger a dummy `bcrypt.compare()` against `DUMMY_BCRYPT_HASH` to equalise response time. Error messages are collapsed to a single generic string in production.
- **Account lockout** — 5 failed attempts triggers a 15-minute lockout. The lockout is also applied on the TOTP step-2 path (`loginWithTotp()`).
- **Password strength** — `ResetPasswordDto` and `ChangePasswordDto` enforce `@MinLength(8)`, one uppercase, one digit, and `@MaxLength(128)`.
- **bcrypt rounds** — `BCRYPT_ROUNDS = 12` is used consistently. No plaintext password comparison anywhere.
- **ProjectAccessGuard IDOR** — Global-role-only assignments (`projectId = null`) do NOT grant project access for non-Admin users. Access requires explicit PM assignment or `ProjectMember` row. Confirmed in `project-access.guard.ts:55-64`.
- **Admin-only eval controller** — `EvalRetrievalController` is decorated with `@UseGuards(JwtAuthGuard, RolesGuard)` + `@Roles('Admin')`. Not accessible by non-admin users.
- **AI token budget** — `assertWithinDailyBudget()` is called before cahier generation. Budget is per-project and configurable via env. No HTTP bypass path found.
- **Backfill CLI not HTTP-accessible** — `backfill-embeddings.ts` is a standalone CLI (`NestFactory.createApplicationContext`), not wired to any HTTP route. Cannot be triggered remotely.
- **File upload security** — Audio uploads: magic-bytes sniffing (`isAudioBuffer`) + MIME whitelist + multer 100 MB limit. Document attachments: extension allowlist + path-traversal containment via `path.resolve()` check + content-type allowlist on download. Upload directory is randomised (`randomUUID()` prefix).
- **SQL injection in pgvector queries** — The production `$queryRawUnsafe` calls in `semantic-tools.ts` and `eval-retrieval.controller.ts` all pass the vector literal as `$1`, `projectId` as `$2`, and numeric parameters as `$3`/`$4`. These are all positional parameters, not string interpolation. The `LIMIT` values are bounded to `1..50` server-side before being passed (line 108 in eval-retrieval, line 105/108 in semantic-tools).
- **CORS configuration** — `CORS_ORIGINS` env var is required in production; validated at startup. Applied to both the HTTP server (`enableCors`) and Socket.IO gateways. Origin defaults to `http://localhost:5173` in dev only.
- **PII redaction before AI calls** — `redactPii()` is applied in the transcription analysis and cahier pipeline before user-supplied text is sent to OpenAI/Gemini. Covers email, IBAN, phone, SIRET, card numbers.
- **XSS on v-html** — Both `v-html` usages (`CahierDocSection.vue` and `MeetingAiPanel.vue`) apply `sanitize()` (DOMPurify with a strict allowlist). The raw content is HTML-escaped before markdown substitution, and DOMPurify is applied as a final defence-in-depth pass. No unprotected `v-html` found.
- **`.env` not in git** — `.gitignore` explicitly lists `.env` and `.env.*` (with the correct exceptions). `git ls-files` confirms the dev `.env` is not tracked.
- **Error message sanitisation in production** — `AllExceptionsFilter` returns only safe, pre-approved error text in production. Stack traces, Prisma codes, and internal error names are suppressed.
- **Docker network isolation** — `postgres` has no `ports:` mapping; it is only reachable within `neoleadge-net`. The NestJS server has no `ports:` mapping. Only the frontend nginx container binds to `127.0.0.1:8002`. The transcription service has no `ports:` mapping.
- **Transcription service authentication** — The FastAPI service requires `X-Transcription-Secret` on every call. Fails to boot if the secret is absent. CORS is locked to the NestJS backend origin. Upload size is double-checked (Content-Length + streaming byte cap). Magic-bytes sniffing applied before processing.
- **Log redaction** — `pinoHttp.redact` strips `Authorization`, `Cookie`, `x-transcription-secret`, `password`, `currentPassword`, `newPassword`, `tempToken`, and `token` fields from all request/response logs.
- **PM self-approval blocked** — `saveFeedback()` in `cahier-des-charges.service.ts` checks `userId === project.projectManagerId` and returns 400.
- **Swagger disabled in production** — `SwaggerModule.setup` is inside `if (!isProduction)`. No API documentation is publicly exposed.
- **Health endpoint information leakage** — In production, `/health` returns only `{ status: 'ok' | 'degraded' }`. Version, uptime, and node version are suppressed.
